Project

Project Proposal

A proposed SLM/LLM system using RamaLama RAG based off Fedora RPM packaging guidelines.

Develop a SLM/LLM using RamaLama RAG based off Fedora RPM packaging guidelines

A Retrieval-Augmented Generation system that ingests Fedora RPM packaging guidelines and serves them through a locally-run small language model via RamaLama.

View Full Proposal →

Problem Statement

In order for a new package to be added to Fedora, it must undergo a formal review. These reviews ensure that the package meets the quality control requirements for Fedora. These reviews rely on manpower and can be really complex. It is also a load of work for the human reviewers as the package has to meet all packaging guidelines. These reviewers might also cause inconsistencies as humans are prone to error.

Project Objectives

Curate and preprocess high-quality dataset of Fedora approved and non-compliant RPM spec files. Build a RAG pipeline that grounds models responses in Fedora documentation guidelines. Use an appropriate SLM/LLM via RamaLama

Technical Approach

This includes; Data Curation and Preprocessing, RAG Pipeline Development, Model Serving with RamaLama, Deployment and interface and Evaluation and testing.

Reflections

What I Experienced

Honest reflections on the challenges I faced and the things I genuinely loved during this journey.

😤

What Challenged Me

  • Transitioning my entire machine from Windows to Fedora in week one it was intimidating navigating an unfamiliar OS ecosystem from scratch while simultaneously diving into tasks.
  • Setting up a Fedora account profile picture was a bit tricky a small thing that turned into a rabbit hole, though it led me to create a video guide helping peers avoid the same frustration.
  • Working with Docling for the first time, especially processing unstructured multilingual scanned PDFs with OCR engines like Tesseract and TesserOCR, threw up several installation and compatibility challenges.
  • Navigating the many Fedora websites and platforms all at once in the first week felt overwhelming before things started to click into place.
  • Balancing peer collaboration, individual tasks, PR reviews, and my own exploration all in week one required careful prioritisation.
💚

What I Loved

  • Getting my very first open-source PR merged on fixing the broken RamaLama docsite logo, was a genuinely proud moment. That feeling of seeing your name in a merged commit is something special.
  • Discovering RamaLama and how effortlessly it runs AI models locally with no configuration. As an AI engineer, watching it handle models I'd previously used in complex setups was really exciting.
  • The collaborative spirit of the intern cohort. Every meeting, whether planned or spontaneous, turned into a genuine learning session. When more peers joined the Tesseract debugging call, it became a really powerful experience as we got to learn from each other.
  • Writing again and creating my very first video. I hadn't posted on my blog or LinkedIn in a while, and this internship gave me the perfect reason to get back into it sharing knowledge about Fedora, Docling, and RamaLama with a real audience. I posted my first video and can't wait to do more.
  • Becoming part of the Fedora community. Fedora has grown on me quickly and it's a community I genuinely want to remain part of.
